  • PostScript マルチプルマスターフォント

    • デザイン上の留意点
    • Type2 (2000年3月, Adobe は Type2 からマルチプルマスターサポートを削除しました)
      • Type2 マルチプルマスターフォーマットの仕様 (廃止された Type2 仕様書の一部)
      • CFF マルチプルマスターフォーマットの仕様 (廃止された CFF 仕様書の一部)

    • PLRM のセクション 5.8.1 には、このフォントフォーマットが文書化されていないと記載されています。
  • PostScript Type42
  • AFM
  • PFM
    • PFM に関する Microsoft の文書は、現在もう見つかりません。このフォーマットは NTF に置き換えられて廃止されたのだろうと思います。
    • 第三者による解説
  • NTF
    • このフォーマットは Windows 2000 以降で PFM ファイルの代わりに用いることになっています。これに関する文書は見つかりませんでした。
  • BDF
    • X11 論理フォント記述子の仕様は X における標準の BDF 属性を定義しています。
    • ABF ― バイナリフォーマット
    • グレイマップをサポートするための BDF の拡張
  • TrueType の規格
    (残念ながら異なるソースごとに、規格のあまり重要でない部分においてわずかに異なる定義を行っている旨、警告しておきます)
    • Apple (私は Apple の散文が難解で、ときには誤解を招くことに気づきました。可能な場合は別の情報源を使用することをお勧めします)
    • Microsoft
    • 雑多な役立つサイト
    • TTC ― TrueType フォントコレクション
  • TrueType への Apple 高度組版機能拡張
  • Apple 変形可能フォント (バリエーションテーブル) ― マルチプルマスターフォントになんとなく似た TrueType における代替品
    • fvar (フォントのバリエーション)
    • gvar (グリフのバリエーション)
    • cvar (cvt のバリエーション)
    • avar (軸のバリエーション)
  • OpenType (TrueType のラッパに埋め込まれた PostScript、または TrueType ラッパに含まれる高度組版機能テーブル)
    • Postscript Type2
    • CFF
    • Adobe 版のファイルフォーマット
      • SING 外字拡張 (より詳しい情報は、グリフレット GDK の documentation サブディレクトリから入手可能です)。
    • MS/Adobe によって指定されていない用字系コードの考えられる情報源: ISO 15924
    • Microsoft の言語/ロケールの全リスト (すべてがサポートされているわけではなく、いくつかは未サポートでしょう)
  • PS Type1 フォントに対する Apple の sfnt ラッパ
  • Postscript Type42 (OpenType の逆で、これは PostScript に埋め込まれた TreuType です)。
  • SVG 1.1 のフォント
    • SVG 1.2 の、フォントヒント指定の提案

  • Windows の各種フォントフォーマット

  • X11 PCF フォーマット
    • 残念ながら本物の規格は存在しません。X11 によるソースコードはここにあります。
    • だから私は自分で説明を書きました……
  • PC スクリーンフォント (psf/psfu/psf2)
  • TeX の各種フォントフォーマット
    • pk パック形式ビットマップフォーマット
    • gf 汎用フォント (ビットマップ) フォーマット
    • tfm メトリックフォーマット
    • これらを読める形にするには、以下のような操作を行う必要があるでしょう:
      $ weave pktype.web
      $ pdftex pktype.tex
  • SIL Graphite フォント (TrueType フォントの洗練された拡張。グリフの文字合成・順序の入れ換え・位置指定などのためのルールを記述する追加テーブル)
  • Palm pilot のフォント (pdb ファイル)
    • フォントレコードフォーマット
    • 残念ながら商用フォーマットなので、サポートの予定はありません。
    • Acorn RISC OS フォントフォーマット (これらのフォントは、しばしば非標準の zip 形式で固められています)。
    • Ikarus IK フォーマットは Peter Karow の著書Digital Formats for Typefaces の Appendices G と I に文書化されています。(現在もコピーを URW++ から入手可能です)
      興味深いことに、曲線の正確なフォーマットは解釈を行うプログラムに任せられています。
    • SFD ファイル (FontForge の内部スプラインフォントデータベースフォーマット)
    • cidmap ファイル (Fontforge の CID→Unicode 対応表のフォーマット)
    • XML 形式の各種フォーマット
      • TTX ―
      • UFO と GLIF ― 統合されたフォントオブジェクトおよびグリフ交換フォーマット
